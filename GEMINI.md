## 🔄 Fluxo de Trabalho (OBRIGATÓRIO)

Toda alteração no repositório deve seguir o fluxo abaixo:

1. **Abrir um Issue** no GitHub descrevendo o que precisa ser feito
   - Referência: https://docs.github.com/en/issues/tracking-your-work-with-issues
2. **Criar uma branch** com nome relacionado ao issue (ex: `fix/textos-50-colunas`)
3. **Fazer as alterações** na branch
4. **Abrir um Pull Request (PR)** referenciando o issue (`Closes #N`)
5. **Revisar e fazer merge** do PR na branch `master`

> ⚠️ NÃO fazer commits direto na branch `master` sem passar pelo fluxo de Issue + PR.
